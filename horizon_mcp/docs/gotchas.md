# HOI4 scripting gotchas

Clausewitz/Jomini engine and scripting-language behaviors that fail silently — no lint error, no error.log entry. Generic to any mod.

- **Temp var name shadowing** — short/generic temp names (`vol`, `ins`, `gap`, `n`...) can collide with engine-reserved identifiers: writes silently no-op, reads return a fixed constant. Always prefix temp names.
- **Scope chain-read returns 0** — `x.CUSTOM_VAR` read through a dynamic scope chain silently returns 0. Fix: `var:x = { set_temp_variable = { PREV.y = CUSTOM_VAR } }`. Also: `on_startup` effects run in scope `None` — anchor with a country/state scope first.
- **Variable-op tooltip param** — variable ops (`set_variable`, `add_to_variable`, etc.) accept `tooltip = <loc key>` with `$LEFT$`/`$RIGHT$` tokens (formattable via `|`). Effects only — `check_variable` ignores it; use `custom_trigger_tooltip` + `[?var]` in triggers instead.
- **check_variable has no >=/<=** — shorthand `var op value` only supports `>`,`<`,`=` (silently wrong if misused, no lint error). Use long form (`var=`, `value=`, `compare=greater_than_or_equals`, etc.) for the rest.
- **scripted_localisation trigger blocks abort on first failed condition** — a `text = { trigger = {...} }` block evaluates its statements in order like an implicit AND-chain; the first failing condition aborts the whole block, so anything sequenced after it — bare effects-in-trigger math included — never runs. `if = { limit = {...} effects }` is just the common way this bites (a failed `limit` makes the `if` itself the failing condition); it's not a special-case bug unique to `if`/`limit`. Fix: compute the value unconditionally upstream, or branch via multiple `text=` entries each gated by their own bare `check_variable` (branch selection, not guarding mid-block). Also: `[?x|%0+]` renders empty — multiply by 100 and use `[?x|0+]%`.
- **scripted_gui triggers block is a trigger scope** — no effect-loops (`for_loop_effect`, `while_loop_effect`, `for_each_loop`) inside `triggers`/`visible`. Allowed: trigger constructs + `set_temp_variable`, `add_to_temp_variable`, `add_to_temp_array`, `custom_effect_tooltip`, `if`/`limit`, `all_of`/`any_of`/`count_of`. To build a temp array of length N, walk an existing array with `all_of`. Exception: `log=` still executes here.
- **effectFile GUI sprites drift off-scale** — sprites using a GUI `effectFile` shader render at a nonstandard, texture-size-dependent scale vs `properties`-positioned elements (no resolution uniform reaches GUI shaders). Bake procedural art into a plain RGBA DDS instead — renders 1:1 at any resolution.
- **global.date is prescaled** — reading `global.date` as a variable returns a scaled value (~0.00001/hour), not hours. Multiply by 50000 first for real time math.
- **iconType properties x/y ignore orientation** — scripted-GUI `properties` x/y on an iconType apply from the parent's top-left, ignoring `orientation = center`. buttonType respects its own orientation. Fix: wrap the icon in a centered `containerWindowType` and position inside that.
